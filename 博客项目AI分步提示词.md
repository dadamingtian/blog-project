# 个人博客项目 AI 分步生成提示词手册

> 用途：这是一份配合《个人博客项目计划书.md》使用的 **分阶段提示词模板**。你可以把对应阶段的提示词逐步发给 AI，让 AI 一步一步完成整个博客项目，而不是一次性输出大量不完整代码。

---

## 1. 使用方式

### 1.1 推荐使用顺序

请严格按下面顺序把提示词发给 AI：

1. 项目总控提示词
2. 第 0 阶段：需求确认与项目边界固化
3. 第 1 阶段：初始化项目骨架
4. 第 2 阶段：后端基础设施搭建
5. 第 3 阶段：管理员登录与鉴权
6. 第 4 阶段：核心数据模型与 API
7. 第 5 阶段：前台博客页面开发
8. 第 6 阶段：后台管理页面开发
9. 第 7 阶段：上传、评论、搜索、归档等补充功能
10. 第 8 阶段：前后端联调与修复
11. 第 9 阶段：测试、优化与验收
12. 第 10 阶段：本地 Git 初始化与提交说明

### 1.2 每次发提示词时的原则

每次只让 AI 完成一个阶段，不要一次让它做完整个项目。

每次都要求 AI：

- 先输出本阶段目标
- 再输出将新增 / 修改的文件树
- 再输出完整代码
- 再说明启动方式
- 再给出自测清单
- 如果上下文过长，优先输出最关键且可直接运行的代码

### 1.3 强烈建议同时附带这份计划书

每次发分阶段提示词时，最好一并告诉 AI：

- 项目计划书是唯一需求基准
- 不允许擅自更换技术栈
- 不允许省略关键模块
- 不允许只给伪代码
- 不允许只讲思路不写代码

## 2. 项目总控提示词

> 这一段建议在正式开始前先发一次，用来统一 AI 的输出风格和工程约束。

```md
你现在是一名资深全栈工程师，请严格按照我提供的《个人博客项目计划书.md》来逐步生成整个项目。

项目技术栈固定如下：
- 前端：Vue 3 + Tailwind CSS + Element Plus
- 后端：Express
- 数据库：MongoDB

你的工作方式必须遵守以下要求：
1. 严格按阶段完成，不要一次性把所有阶段混在一起。
2. 每个阶段都要先说明目标和完成范围。
3. 每个阶段都要先输出文件结构变化，再输出完整代码。
4. 输出的代码必须尽量完整、可运行，不能只给片段或伪代码。
5. 命名要统一，目录职责要清晰。
6. 前后端都要有基础错误处理。
7. 后台接口必须有鉴权。
8. 代码风格要适合真实项目，而不是教学 demo。
9. 如果一个阶段内容过多，请优先保证代码可运行，并明确说明下一阶段继续补齐哪些内容。
10. 除非我明确要求，否则不要擅自更换技术栈、状态管理方案、UI 库或数据库。
11. 所有接口都请统一返回格式，例如：{ code, message, data }。
12. 所有表单都请同时考虑前端校验和后端校验。
13. 完成每个阶段后，请给出运行步骤和本阶段测试方法。
14. 当我发送“继续下一阶段”时，你再进入下一阶段。

现在请等待我发送具体阶段任务。
```

## 16. 最推荐的实际使用方法

最稳妥的方式是：

1. 先把《个人博客项目计划书.md》发给 AI
2. 再发“项目总控提示词”
3. 再从第 0 阶段开始，一阶段一阶段推进
4. 每完成一个阶段，本地先落地代码并运行测试
5. 确认无误后，再让 AI 继续下一阶段
6. 最终测试通过后，再按第 10 阶段说明初始化本地 Git 仓库并提交

这样做的好处是：

- AI 不容易一次性输出失控
- 每一步都更容易检查
- 更适合真实项目逐步落地
- 出现问题时更容易定位是哪一阶段引入的
