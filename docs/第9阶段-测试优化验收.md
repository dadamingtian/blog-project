# 第 9 阶段：测试、优化与验收（可执行版）

本文档用于项目联调后的测试与上线前验收准备。

---

## 0. 测试前准备

1. 启动 MongoDB。
2. 启动后端：

```bash
cd backend
npm run dev
```

3. 初始化管理员（首次）：

```bash
cd backend
npm run admin:init
```

4. 启动前端：

```bash
cd frontend
npm run dev
```

5. 确认环境变量：
- 前端 `VITE_API_BASE_URL` 指向正确后端地址
- 后端 `CORS_ORIGINS` 包含前端访问地址

## 1. 后端接口测试建议（可执行）

### 1.1 冒烟测试（推荐先跑）

```bash
cd backend
npm run test:smoke
```

覆盖点：
- 健康检查
- 管理员登录与鉴权
- 分类/标签/文章 CRUD 核心链路
- 评论提交 + 审核 + 前台可见
- 上传接口
- 仪表盘接口

### 1.2 手工 API 回归（建议用 Apifox / Postman）

#### A. 鉴权
1. `POST /api/auth/login` 正确账号密码 -> `code=0`
2. 错误密码 -> 401 + 统一错误结构
3. `GET /api/auth/profile` 无 token -> 401
4. `GET /api/admin/access-check` 带 token -> 通过

#### B. 文章
1. 创建草稿文章（有分类）
2. 编辑文章为 published
3. 前台 `GET /api/articles` 能看到 published
4. 前台 `GET /api/articles/:slug` 返回详情 + 上一篇/下一篇
5. 删除文章后前台访问详情返回 404

#### C. 分类/标签
1. 创建时不传 slug，后端自动生成
2. 重名 slug 自动追加后缀确保唯一
3. 被文章引用时删除分类/标签应返回 409

#### D. 评论
1. 前台提交评论
2. 后台评论列表可查到
3. 审核通过后前台评论列表可见
4. 审核拒绝后前台不可见

#### E. 站点配置
1. `GET /api/site/config`
2. 后台更新 `siteName / aboutContent / avatar / socialLinks`
3. 前台刷新后信息同步

#### F. 上传
1. `POST /api/admin/upload/image` 上传 jpg/png/webp/gif 成功
2. 上传超 2MB 或非图片类型返回错误
3. 返回 `url/path` 可直接访问

## 4. 常见异常场景测试清单

1. token 过期/伪造
- 预期：后端 401，前端清理登录态并跳转登录页

2. 参数非法
- 例如 `page=-1`、`pageSize=abc`
- 预期：后端不崩溃，返回校验错误或自动回退默认分页

3. 重复 slug
- 预期：自动生成唯一 slug（如 `xxx-2`）

4. 分类/标签删除冲突
- 被文章引用时删除
- 预期：409 + 明确提示

5. 上传非法文件
- 非图片、超大小
- 预期：415/400 且提示清晰

6. MongoDB 暂时不可用
- 预期：接口失败时返回统一错误结构

## 5. 性能与代码结构优化建议（落地级）

1. 前端代码分包
- 现状：主包较大（编辑器 + 图表）
- 建议：对管理端路由做动态导入；`vite manualChunks` 拆 `echarts` 与 `toast-ui-editor`

2. 列表查询索引
- `Article(status, publishedAt, createdAt)` 已有
- 建议补充：`Comment(article, status, createdAt)` 并定期检查慢查询

3. 仪表盘聚合缓存
- 建议：给 `/admin/dashboard/overview` 增加短缓存（30~60s）

4. 上传目录治理
- 建议：定期清理孤儿文件；生产迁移到对象存储

---

## 6. 安全性基础检查

1. JWT
- 使用强随机 `JWT_SECRET`
- 生产禁用默认密钥

2. CORS
- 严格配置 `CORS_ORIGINS`，不使用 `*`

3. 输入校验
- 前后端均做校验（当前已覆盖核心表单）

4. 上传安全
- 仅允许白名单图片类型 + 大小限制

5. Helmet
- 已启用，建议结合反向代理完善安全头策略

6. 权限边界
- `/api/admin/**` 必须走鉴权中间件（当前已接入）

## 9. 生产环境配置建议

### 后端
- `NODE_ENV=production`
- `JWT_SECRET=<高强度随机串>`
- `CORS_ORIGINS=https://your-domain.com`
- `PUBLIC_SERVER_URL=https://api.your-domain.com`
- 使用 PM2 或 systemd 托管进程

### 前端
- `VITE_API_BASE_URL=https://api.your-domain.com/api`
- 使用 Nginx 托管静态资源并开启 gzip/br

### 数据库
- 创建专用低权限账号
- 定期备份 + 恢复演练
